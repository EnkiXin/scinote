## 📖 Notation & Conditions

### Task Levels (ExpVid paper)

| Code | Level | Tasks | Video length |
|---|---|---|---|
| **L1** | Fine-grained Perception | materials / tools / operation / quantity | ~8s clips |
| **L2** | Procedural Understanding | sequence_generation / sequence_ordering / step_prediction / video_verification | ~48s stages |
| **L3** | Scientific Reasoning | experimental_conclusion / scientific_discovery | ~8min full videos |

### Conditions (current 5-condition unified design, [evaluate_unified.py](evaluate_unified.py))

All conditions share the **same answer prompt template**, only the "context block" before the question varies → isolates input modality from prompt phrasing.

| Code | Input | Purpose |
|---|---|---|
| **C0** | video only | Standard baseline (≡ ExpVid paper's video-only condition) |
| **C1** | note only, no video (Design X) | Is the note a lossless encoding of the video? |
| **C2** | video + note (Design Y) | Does the note help as augmentation? |
| **C3** | video + random_note (other video's same-task note) | Isolate **note content** vs **note format scaffold** (paper-defending control) |
| **C4** | video + ASR | Upper bound reference (ASR is roughly the labels that drove ExpVid annotation) |

The note is generated by Stage 1 of [evaluate_twostage_v2.py](evaluate_twostage_v2.py)'s **task-aware** prompts; it's cached on disk so C1/C2/C3 reuse the same notes per (task, video).

## 🔑 Headline Findings (latest H200 data)

### Finding 1 — Phase 0 baseline reproduces paper Table 2

H200 Qwen2.5-VL-7B C0 baseline matches paper within ~3pp on most tasks. Pipeline is correctly reproduced (see [Phase 0 table](#phase-0--c0-baseline--v2-with-old-prompt-on-h200)).

### Finding 2 — ASR leakage on L1 is huge (~+50-60pp)

C4 (video + ASR) on L1 is **~90-97% accuracy** vs C0 video-only ~34-65%. Confirms the original ExpVid paper's annotation pipeline extracts labels from ASR, so adding ASR to the prompt nearly solves L1. This is **not a contribution of our method** — it's a property of the ExpVid benchmark design.

### Finding 3 — Two-stage notes ≠ "free improvement" on L1

| L1 task | C0 video | C1 note only | C2 video+note | C3 video+random_note |
|---|---|---|---|---|
| materials | 34.0 | **31.7** (-2.3) | **36.7** (+2.7) | 35.0 (+1.0) |
| tools | 36.3 | 31.2 (-5.1) | 35.2 (-1.1) | 34.5 (-1.8) |
| operation | 64.6 | 51.0 (-13.6) | 57.2 (-7.4) | **62.2** (-2.4) ⚠️ |
| quantity | 47.2 | 34.1 (-13.1) | 40.8 (-6.4) | TBD |

- **C1 (note replacing video) consistently loses on L1**: -2.3 to -13.6pp across 4 tasks → note is a **lossy compression** when the task needs high-fidelity visual reading
- **C2 (video + note) mixed**: helps materials (+2.7) but hurts other 3 L1 tasks → augmentation isn't always free
- **⚠️ Surprising C3 control**: on operation, C3 (random other-video note) BEATS C2 (real note) by +5pp. **If this holds at full n=938, it suggests "prompt scaffold of having structured text" matters more than the note's actual content on this task** — a paper-defending finding that any improvement is at least partly prompt-engineering, not note semantics.

### Finding 4 — L1 disagreement analysis (1238 samples, per-sample CSV)

[`results_h200_unified/analysis_l1_materials.csv`](results_h200_unified/analysis_l1_materials.csv) + [`analysis_l1_materials_summary.md`](results_h200_unified/analysis_l1_materials_summary.md):

| Pattern | n | % | Interpretation |
|---|---|---|---|
| Only ASR correct (C4✓, others✗) | 543 | 43.9% | ASR leakage zone |
| All 4 conditions agree (correct) | 234 | 18.9% | Easy cases |
| Mixed | 180 | 14.5% | — |
| C0 correct, C1 wrong | 95 | 7.7% | Note paraphrase loses video info |
| All 4 wrong | 86 | 6.9% | Hard cases |
| C1 alone > C0 (note > video) | **57** | **4.6%** | Note's labels disambiguate options |
| C2 > C0 (video+note > video) | **43** | **3.5%** | Note adds disambiguating signal |

Concrete patterns from the CSV examples:
- ✅ Note helps when it captures **explicit labels** ("AEROSOLIC CLASSIFIER" gives domain, "Culture conditions: 28 °C 5% CO2" gives context)
- ❌ Note hurts when it focuses on **containers/tools** while the question asks about the **substance** (Olympus microscope noted; cells in the field missed)
- → **Implication for paper**: task-aware prompt is still too generic; need to train the model to focus on the question's actual referent (this is the motivation for counterfactual SFT for note-taking)
